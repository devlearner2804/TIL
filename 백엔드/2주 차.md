# 객체지향 (OOP, Object-Oriented Programming)

객체지향 프로그래밍은 프로그램을 객체(Object)라는 단위로 구성하여 개발하는 방식이다. 객체는 데이터(속성)와 기능(메서드)을 하나로 묶은 개념이다.

객체지향의 주요 특징은 다음과 같다.

첫째, 캡슐화는 데이터와 기능을 하나로 묶고 외부에서 직접 접근하지 못하도록 보호하는 것이다.

둘째, 상속은 기존 클래스의 기능을 새로운 클래스가 물려받아 재사용할 수 있도록 하는 것이다.

셋째, 다형성은 하나의 이름을 가진 기능이 상황에 따라 다르게 동작하는 특성이다.

# HTTP / HTTPS 개념 및 차이점

HTTP(HyperText Transfer Protocol)는 웹에서 클라이언트와 서버가 데이터를 주고받기 위한 통신 규칙이다. 하지만 데이터가 암호화되지 않아 보안에 취약하다.

HTTPS(HyperText Transfer Protocol Secure)는 HTTP에 SSL/TLS 보안 기능을 추가한 프로토콜이다. 데이터를 암호화하여 전송하므로 보안성이 높다.

HTTP는 기본적으로 80번 포트를 사용하고, HTTPS는 443번 포트를 사용한다. 또한 HTTP는 속도가 빠르지만 보안이 약하고, HTTPS는 약간의 속도 저하가 있지만 안전하다.

# HTTP 메서드 정리 
HTTP 메서드는 클라이언트가 서버에 요청할 때 사용하는 방식이다.

GET: 서버에서 데이터를 조회할 때 사용한다.

POST: 서버에 데이터를 생성하거나 전송할 때 사용한다.

PUT: 기존 데이터를 전체 수정할 때 사용한다.

DELETE: 서버의 데이터를 삭제할 때 사용한다.

PATCH: 데이터의 일부만 수정할 때 사용한다.

이러한 메서드를 통해 서버와 클라이언트 간 다양한 작업이 이루어진다.

# RESTful API 개념
RESTful API는 REST 원칙을 따르는 API 설계 방식이다. REST는 웹에서 자원을 URL로 표현하고 HTTP 메서드를 이용해 자원을 처리하는 구조이다.

RESTful API의 특징은 다음과 같다.
첫째, 자원을 URL로 표현한다. 예를 들어 /users/1은 특정 사용자를 의미한다.
둘째, HTTP 메서드를 사용하여 자원을 처리한다.
셋째, 서버는 클라이언트의 상태를 저장하지 않는 무상태 구조를 가진다.

이 방식은 구조가 단순하고 확장성이 좋기 때문에 많이 사용된다.

# 무상태성
무상태성은 서버가 클라이언트의 이전 요청 상태를 저장하지 않는 특성을 의미한다.

즉, 각 요청은 독립적으로 처리되며 서버는 이전 요청을 기억하지 않는다. 따라서 클라이언트는 매 요청마다 필요한 정보를 모두 포함해야 한다.

무상태성의 장점은 서버 구조가 단순하고 확장성이 높다는 것이다. 반면 단점은 매 요청마다 정보를 다시 전달해야 하므로 비효율이 발생할 수 있다는 점이다.

무상태성은 REST API 설계의 중요한 특징 중 하나이다
